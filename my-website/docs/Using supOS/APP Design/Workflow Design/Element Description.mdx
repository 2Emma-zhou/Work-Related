### Timer Event

<div>
        <div className="p">
          A timer event sets the deadline time of current task or the last task.
          It can be used in following ways:
          <ul id="concept_iml_dwt_r5b__ul_nf1_jxt_r5b">
            <li>
              Place it between two tasks, and the second task only starts when
              reaching the set time.
            </li>
            <li>
              Place it on a task, and if the task is finished before the set
              time, the timer becomes invalid; if not, a branch task is
              triggered.
            </li>
          </ul>
        </div>

## Subprocess

<p>
      A subprocess is a part of a workflow. It is generally started with a
      signal event and triggered by signals thrown by multiple tasks in main
      process.
    </p>

## Gateway

<div>
      <p>
        Gateway decides how subsequent process is proceeded based on different
        gateway types.
      </p>
      <ul id="gateway__ul_qs3_gs5_r5b">
        <li>
          <strong>Exclusive</strong>: Execute only one subsequent branch process
          that meets the requirements.

:::info

Branches after an
            exclusive gateway must be configured with conditions.

:::

</li>
        <li>
          <strong>Parallel</strong>: Execute all subsequent branch processes.

:::info

Before an inclusive

:::

</li>
        <li>
          <strong>Inclusive</strong>: Execute subsequent branch processes that
          meet the requirements and processes that are not restricted with any
          conditions.
        </li>
      </ul>
      <figure className="fignone" id="gateway__fig_jvp_ms5_r5b">
        <img id="gateway__image_pyy_ms5_r5b" src="https://wordpressfreezonex.oss-accelerate.aliyuncs.com/workflow/28.png" />
      </figure>
      <p>
        Click the connection lines after the gateway to set conditions of
        corresponding branch.
      </p>
      <ul id="gateway__ul_hfb_f55_r5b">
        <li>
          <strong>Assignee</strong>: Allows you to select a receiver for the
          subsequent process on the linked page of last step.
        </li>
        <li>
          <strong>Expression</strong>: Set conditions to decide whether to
          execute subsequent processes.
        </li>
      </ul>
      <figure className="fignone" id="gateway__fig_vhl_d55_r5b">

## End

<div>
      <p>The element that ends the workflow.</p>
      <ul id="end__ul_ohs_jv5_r5b">
        <li>
          <strong>End event</strong>: End the current branch process.
        </li>
        <li>
          <strong>Terminate event</strong>: Terminate the whole workflow. When
          used in a sub-process, enable <strong>Terminate all</strong> to end
          the whole process. Otherwise, only the sub-process is ended.
        </li>
