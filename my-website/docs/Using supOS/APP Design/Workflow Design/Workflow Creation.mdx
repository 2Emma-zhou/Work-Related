## Creating Workflow


<div>
      <section className="section" id="id_gkp_lsp_q5b__context_zm3_cnq_q5b">
        Draw a workflow to demonstrate actual business flow with related pages
        and personnels.
      </section>
      <section id="id_gkp_lsp_q5b__steps_eqj_hnq_q5b">
        <ol id="id_gkp_lsp_q5b__steps_eqj_hnq_q5b">
          <li className="stepexpand">
            <span>
              In <strong>Business Designer</strong>, click the{" "}
              <strong>Workflow Management</strong> tab.
            </span>
          </li>
          <li className="stepexpand">
            <span>
              Click <strong>New</strong>, enter workflow information and then
              click <strong>Confirm</strong>.
            </span>
          </li>
          <li className="stepexpand">
            <span>
              Click the <strong>Start Flow</strong> icon, and then click{" "}
              <img id="id_gkp_lsp_q5b__image_xvc_5nq_q5b" src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/workflow/5.png" />{" "}
              to add the first step.
            </span>
            <div>
              In this example, the first step is to add a new sales order.
              <ol type="a" id="id_gkp_lsp_q5b__ul_nxb_c4q_q5b">
                <li>
                  Click the first step, and then enter the{" "}
                  <strong>Step name</strong> on the right side.
                </li>
                <li>
                  Set <strong>Node type</strong> to <strong>User task</strong>.
                </li>
                <li>
                  Click the frame next to <strong>Page</strong>, and then select
                  the <strong>Add Order</strong> page you just created.
                </li>
                <li>
                  Click <strong>Config</strong> next to{" "}
                  <strong>Executor</strong>, select a personnel as the task
                  executor.
                  <p>In this example. the initiator is set as executor.</p>
                </li>
              </ol>
            </div>
            <div>
              <figure className="fignone" id="id_gkp_lsp_q5b__fig_bcb_54q_q5b">
                <img id="id_gkp_lsp_q5b__image_ccb_54q_q5b" src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/workflow/6.png" />
              </figure>
            </div>
          </li>
          <li className="stepexpand">
            <span>
              Click the first step and then click{" "}
              <img id="id_gkp_lsp_q5b__image_n5v_2pq_q5b" src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/workflow/5.png" />{" "}
              to add a next step.
            </span>
            <div>
              In this example. the next step is to check and approve the order
              information.
              <ol type="a" id="id_gkp_lsp_q5b__ol_x1n_hpq_q5b">
                <li>Set the step name and node type.</li>
                <li>Select the same page as the first step.</li>
                <li>
                  Select an executor.
                  <p>
                    In this example, the initiator is selected for easy
                    operation.
                  </p>
                </li>
                <li>
                  (Optional) Under <strong>Service</strong>, select a service
                  from an instance to add the approved sales order data to the
                  database.

:::info

The workflow can proceed without binding a service, and no
                    data will be added to the database.

:::

</li>
              </ol>
            </div>
            <div>
              <figure className="fignone" id="id_gkp_lsp_q5b__fig_ckh_hrq_q5b">
